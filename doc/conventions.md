# Conventions — правила разработки

Источник истины по стеку, границам MVP и pipeline: [vision.md](vision.md)
Идея продукта: [00_project_idea.md](00_project_idea.md)

Ниже — только правила, влияющие на качество кода. Детали стека и данных — в vision.md, не дублируем.

---

## Границы

- Реализуем **только MVP** из vision.md. Новые фичи, библиотеки и слои — только после явного запроса.
- Запрещено добавлять зависимости из раздела «Не используем» в vision.md.
- Минимальный diff: не рефакторить и не «улучшать» код вне текущей задачи.

---

## Архитектура кода

- **Offline / online:** индекс строит `scripts/build_index.py`; Streamlit только загружает индекс и отвечает.
- **Один модуль — одна роль:** ingest, chunking, index, retrieval, demo-answer, UI — отдельные файлы, без god-objects.
- **Конфиг в одном месте** (`app/config.py`): пути, `top_k`, размер чанка, стоп-слова, имена файлов индекса.
- Пути — через `pathlib.Path`, не строковая склейка.

---

## RAG-поведение (обязательно в коде)

- Каждый ответ содержит **источники:** `doc_id`, заголовок, фрагмент текста, score.
- Demo-режим: ответ **только из найденных чанков**; если контекста нет (score ниже порога) — явный отказ, без выдуманных фактов.
- Retrieval возвращает **top-k** с score; UI показывает найденные фрагменты до ответа.
- Индексируется **только текст** статьи; метаданные (рубрика, дата, url) — для источников и трассировки.

---

## Стиль и читаемость

- KISS: plain functions и простые dict, без лишних абстракций, фабрик и DI.
- Код и идентификаторы — на английском; комментарии и пользовательские сообщения — на русском, где нужно для учебности.
- Без «магии»: явные шаги pipeline лучше, чем скрытая логика фреймворка.
- Обработка ошибок — только там, где без неё ломается UX (нет индекса, пустой вопрос); без blanket try/except.

---

## Тесты и проверка

- **pytest:** chunking, retrieval (k результатов), наличие `doc_id` в источниках, корректный отказ.
- Не писать тесты ради покрытия; каждый тест проверяет реальное поведение MVP.

---

## Зависимости и окружение

- **Python 3.10+**, менеджер окружения — **uv**; venv в `.venv/` (не коммитить).
- Зависимости — в `pyproject.toml`; установка: `uv sync`. Подготовка данных — `uv sync --extra data`.
- Не использовать pip/poetry/conda напрямую.
- Секреты и API-ключи не добавлять: demo-режим без внешних LLM.
- `data/index/` и `data/processed/` — не коммитить (`.gitignore`); `data/raw/datasets.json` — коммитим.
