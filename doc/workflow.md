# Workflow — как выполнять работу по тасклисту

**Источники:** [tasklist.md](tasklist.md) · [conventions.md](conventions.md) · [vision.md](vision.md)

---

## Цикл одной итерации

```
Реализация → Проверка → Обновление tasklist → Коммит → Следующая итерация
```

Работать **строго по tasklist.md** — одна итерация = один проверяемый результат.

---

## Шаг 1. Реализация

- Код по conventions.md и vision.md.
- Только задачи **текущей** итерации.
- Минимальный diff, KISS.

## Шаг 2. Проверка

- Выполнить команду **«Проверка»** из tasklist.md для этой итерации.
- Зафиксировать результат (успех / ошибка).
- При ошибке — исправить в рамках той же итерации, не переходить дальше.

## Шаг 3. Обновление tasklist.md

- В таблице прогресса: статус **✅**, обновить «Готовность MVP».
- Отметить чекбоксы `- [x]` выполненных задач.

## Шаг 4. Коммит

```bash
git add .
git commit -m "iter N: краткое описание итерации"
```

Один коммит = одна итерация.

## Шаг 5. Переход дальше

Вернуться к шагу 1 для следующей итерации.

---

## Запрещено

- Пропускать итерации или объединять несколько без необходимости.
- Менять scope MVP без обновления vision.md.
- Коммитить непроверенную итерацию.
